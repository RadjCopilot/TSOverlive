# Иконка в Dock и трее

## Текущее поведение

✅ **Иконка в трее**: Видна всегда, пока приложение запущено  
✅ **Иконка в Dock**: Видна, пока приложение запущено

## Почему приложение в Dock?

В Tauri 1.x нет простого способа скрыть приложение из Dock, но оставить иконку в трее видимой.

### Попытки решения:

1. ❌ `ActivationPolicy::Accessory` - скрывает и Dock, и трей
2. ❌ `ActivationPolicy::Prohibited` - блокирует весь UI
3. ❌ `skipTaskbar: true` - скрывает трей когда окно неактивно
4. ❌ `LSUIElement` в Info.plist - не поддерживается в Tauri 1.x конфиге

### Решение для Tauri 2.x

В Tauri 2.x можно использовать:

```json
"bundle": {
  "macOS": {
    "infoPlist": {
      "LSUIElement": true
    }
  }
}
```

Это скроет приложение из Dock, но оставит трей.

## Текущая конфигурация

```json
"skipTaskbar": false,
"systemTray": {
  "iconPath": "icons/tray-icon.png",
  "iconAsTemplate": true
}
```

Это обеспечивает:
- ✅ Трей всегда виден
- ✅ Меню трея работает
- ✅ Клик по трею показывает/скрывает окно
- ⚠️ Приложение видно в Dock

## Альтернатива

Если критично скрыть из Dock - нужно обновиться до Tauri 2.x.

Для текущей версии (Tauri 1.5) - приложение будет в Dock и трее одновременно.
